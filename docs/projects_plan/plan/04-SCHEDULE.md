# KT CloudNative Study 상세 일정

## 문서 목적

이 문서는 `03-MILESTONES.md`의 마일스톤을 날짜 기준으로 빠르게 확인하기 위한 일정표다. 범위와 성공 기준은 `02-PROJECT_PLAN.md`, 마일스톤별 산출물은 `03-MILESTONES.md`, 실제 작업 단위는 `../workplans/*.yaml`을 기준으로 한다.

## 핵심 날짜

| 날짜 | 이벤트 | 목표 |
| --- | --- | --- |
| 2026-05-27(수) | 계획 시작 | PRD, MVP 범위, 시나리오, Epic, workplan을 정렬한다. |
| 2026-06-02(화) | Phase 0 종료 | P0 시나리오, 역할, Ready 기준, 작업 의존관계를 고정한다. |
| 2026-06-09(화) | Sprint 1 종료 | AWS/GitOps 경로와 정상 사용자 흐름 기준선을 확인한다. |
| 2026-06-12(금) | 중간 발표 | 방향, 기본 상태, 초기 검증 결과, 남은 위험 요소를 공유한다. |
| 2026-06-16(화) | Sprint 2 종료 | 관측성, 통신 제어, 보안 스캔 중 핵심 검증 결과를 확보한다. |
| 2026-06-19(금) | 실험 결과 고정 | 최종 발표에 사용할 로그, 지표, 캡처, 테스트 결과를 고정한다. |
| 2026-06-22(월) | 발표 준비 시작 | 새 실험 없이 자료 정리, 흐름 구성, 백업 캡처를 진행한다. |
| 2026-06-24(수) | 최종 발표 | 문제, 목표, 설계, 검증, 결과, 회고를 연결해 발표한다. |

## 구간별 초점

| 구간 | 기간 | 초점 | 완료 기준 |
| --- | --- | --- | --- |
| Phase 0 | 2026-05-27 ~ 2026-06-02 | 범위, 시나리오, Epic, workplan 확정 | P0와 선택 과업이 분리되고 작업 의존관계가 정리된다. |
| Sprint 1 | 2026-06-03 ~ 2026-06-09 | 기준선과 E2E 확보 | 정상 사용자 흐름과 AWS/GitOps 기본 경로가 한 번 이상 확인된다. |
| Sprint 2 | 2026-06-10 ~ 2026-06-16 | 운영 기능과 정책 검증 | 장애, 보안, 통신 차단 중 최소 1개 이상 실제 결과가 있다. |
| Sprint 3 | 2026-06-17 ~ 2026-06-19 | 실험 결과 고정 | 발표에 사용할 운영 검증 결과가 캡처와 함께 확정된다. |
| Final Prep | 2026-06-22 ~ 2026-06-24 | 발표 준비와 리허설 | 최종 발표 흐름, 백업 자료, 회고가 준비된다. |

## 사용 방법

- 일정 변경은 먼저 이 문서에 반영하고, 필요한 경우 `03-MILESTONES.md`의 통과 기준을 갱신한다.
- 작업 추가나 우선순위 변경은 `../workplans/*.yaml`에서 수정한다.
- 중간 발표 이후에는 새 범위를 늘리기보다 2026-06-19까지 검증 가능한 항목을 닫는 데 집중한다.
